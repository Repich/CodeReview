# Нормы для длительных операций и фоновых заданий

Используется как краткий контекст для LLM при поиске нарушений, связанных с длительными серверными вызовами, импортом файлов и обработкой больших объёмов данных.

## Ключевые выдержки из norms.yaml

1. **Избегать длительных клиент→сервер вызовов**  
   Источник: `norms.yaml:46986-47034`.  
   - Любой серверный вызов, который может выполняться более 8 секунд, следует выполнять асинхронно (фоновое задание).  
   - К длительным операциям относятся загрузка/выгрузка данных, заполнение больших табличных частей, обращение к файловой системе, сетевые вызовы.

2. **Общий подход к асинхронному выполнению**  
   Источник: `norms.yaml:47010-47073`.  
   - Алгоритм длительной обработки располагается в модуле менеджера/общем модуле и запускается фоном.  
   - Пользовательский интерфейс показывает индикатор и отслеживает состояние через `ДлительныеОперации`.  
   - Результаты помещаются во временное хранилище, клиент опрашивает состояние и забирает результат.

3. **Ожидание и информирование пользователя**  
   Источники: `norms.yaml:2200-2217`, `norms.yaml:36066-36190`.  
   - При старте длительной операции необходим индикатор/сообщение.  
   - Для отчётов и обработок используется стандартный механизм ожидания длительных операций.

4. **Запрет длительных транзакций**  
   Источники: `norms.yaml:13922-13940`, `norms.yaml:15866-15870`.  
   - В транзакциях запрещено выполнять длительные циклы, чтение файлов, тяжёлые вычисления.  
   - Долгие транзакции приводят к блокировкам; нужно дробить процесс либо выносить в фон.

5. **Работа с файлами и внешними ресурсами**  
   Источники: `norms.yaml:13995-14010`, `norms.yaml:27973-27980`.  
   - Чтение файлов, обращение к сетевым ресурсам и сервисам считается потенциально длительным и должно выполняться вне интерактивного запроса.  
   - Для импорта/экспорта рекомендуется использовать фоновые задания и временное хранилище.

## Ожидаемые выводы от LLM

- Если клиентский обработчик (`&НаКлиенте` команда) синхронно вызывает серверный метод, в котором:
  - читаются или пишутся файлы (`ДиалогВыбораФайла`, `ОписаниеПередаваемогоФайла`, `ПолучитьИзВременногоХранилища`),
  - выполняется `Запрос` с `ПОМЕСТИТЬ`, вложенные циклы, запись объектов,
  - отсутствует механизм `ДлительныеОперации.*`,
  
  → LLM должна сформулировать предложение новой нормы или сослаться на пункты выше, требуя вынести операцию в фоновое задание.

- Если обработка результата потенциально долгая (заполнение табличных частей, массовая запись справочников) в интерактивном сценарии, необходимо рекомендовать:
  1. Обернуть логику в фоновое задание.  
  2. Использовать временное хранилище и индикатор ожидания.  
  3. Возвращать результат порциями, не блокируя UI.

## Формат ответа модели

Просим LLM возвращать список объектов вида:

```json
{
  "norm_id": "LONG_SERVER_OPS_ASYNC",
  "section": "Длительные операции",
  "category": "performance_runtime",
  "norm_text": "…краткое описание нарушения…",
  "source_reference": "norms.yaml:46986-47073",
  "evidence": [
    {
      "file": "CommonModules/DangerousModule.bsl",
      "lines": "200-250",
      "reason": "Синхронная загрузка Excel и запись объектов без фонового задания"
    }
  ]
}
```

Задача оператора — перенести новые нормы в `norms.yaml` и при необходимости реализовать детектор.
