# План дальнейших работ

1. **Авторизация и личный кабинет**
   - Перейти на внешний IdP/OIDC (минимум — refresh tokens и парольные политики).

   - Поддержка тарифов и ограничений «N запусков в месяц».

2. **Очередь и worker**
   - Заменить HTTP polling на Redis queue (BullMQ/Sidekiq-подобный pipeline).
   - Добавить retry/backoff, trace-id, Prometheus метрики (детектор, длительность, статус LLM).
   - Для LLM сделать кэширование повторных подсказок по `file_hash + norm_version`.

3. **Детекторы и аналитика**
   - Продолжить пополнение `norms.yaml` (новые разделы из «Системы стандартов» и «Руководства разработчика»).
   - Встроить AST-парсер для популярных паттернов (длительные операции, работа с регистрами).
   - Собрать TP/FP статистику (feedback от ревьюеров) и автоматические отчёты.
   - Добавить экспорт findings в DOCX/PDF и интеграцию с downstream системами.

4. **Backend / API**
   - Дополнить `/api/admin/access-logs` фильтрами (по статусу, роли), добавить очистку/архивацию.
   - Реализовать REST для IOLog (список, удаление, повторная выдача файлов), добавить подписанные ссылки.
   - Включить rate limiting на уровне backend (помимо Caddy) и вынести настройки блок-листов в БД.

5. **UI**

   - Показать счётчик времени/статуса прямо в списке запусков, сделать уведомления о завершении анализа.
