# Changelog

## 2026-01-27
- Добавлена настройка пользователя «Использовать все нормы».
- В пайплайн LLM добавлены отдельные шаги для критических норм и норм из norms.yaml.
- Результаты паттернов/критических/обычных норм теперь объединяются в LLM с удалением дублей.
- Артефакты LLM получили новые типы стадий: critical, norms, merge.
- В UI добавлен просмотр артефактов (.txt/.json) в модальном окне с подсветкой JSON.
- Добавлена статическая норма: &НаСервере без обращения к Объект/ЭтаФорма/ЭтотОбъект.

## 2026-01-22
- Добавлены настройки интерфейса (раздельно/вместе).
- Добавлено окно с changelog в личном кабинете.
- В объединённом режиме статические нарушения показываются в левом списке с LLM.
- Клик по статической карточке подсвечивает соответствующий код.
- Добавлена маскировка строковых литералов перед отправкой в LLM с отчётом в артефактах.
- Добавлено маскирование комментариев перед отправкой в LLM (отчёт в артефактах).
- Добавлено правило: не более 5 параметров у процедуры/функции (minor).
- Добавлена настройка пользователя для отключения проверки паттернов.
- Добавлены ручные нормы CUSTOM_003..CUSTOM_006 (безопасные расширения, уникальные ключи, проверки структуры).
- LLM теперь сначала выбирает релевантные нормы по названиям, затем проверяет только их.
- Лимит выбора норм увеличен до 40k символов; артефакты шага выбора помечаются как select.
